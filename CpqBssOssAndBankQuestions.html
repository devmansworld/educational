Introduction & Scope

This document condenses the CPQ, BSS and OSS knowledge you will need for practical telecom certification: product configuration syntax (test-system aligned), pricing mechanics, lifecycle management, integration endpoints, and rapid-revision multiple-choice questions grouped by topic.

Core Concepts

  • CPQ: Configure-Price-Quote — configures products, enforces rules, applies pricing, emits quotes/orders.
  • BSS: Business Support Systems — customer-facing (CRM, Order Mgmt, Billing, Subscription).
  • OSS: Operations Support Systems — network-facing (Provisioning, Activation, Fault/Perf mgmt).
  • Product Catalog: Single Source of Truth (Sigma Catalog / Catalog service) consumed by CPQ and order systems.
  • Elastic API: REST endpoints exposing catalog, baskets, orders, subscriptions, accounts for integration and automation.

Test-System Syntax & Key Expressions

This section reflects the exact syntax your test system expects — memorise these forms for the exams.

  • Parent Attribute Reference: {$Parent:Attribute_Reference_0} — display value of an attribute from the parent product on a related product.
  • Dynamic Lookup: {$Lookup(QueryName):Attribute_Reference_0} — retrieve dynamic values from a query/catalog.
  • $Linked: $Linked — reference to the parent $Object of the current basket (test-system behaviour).
  • Other: $Object (current basket object), $Parent (parent product reference), $Lookup(Query) (alternate query form).

Pricing Models & Actions

  • Pricing Models: base price, list price, calculated price, rule-based pricing.
  • Exam note: Calculated price is often used only for demos/prototyping in test scenarios.
  • Advantages: Pricing models allow business users to modify price matrix without IT; product managers can manage changes (test answer: both).
  • Discount action (exam): Use set discount type in combination with set discount value to calculate discounted amounts.

MCQs — CPQ / CloudSense (grouped)

Q1. Which expression is used to display the value of an attribute from the parent product on a related product?

  1. {(Parent):Attribute Name}
  2. {$Parent:Attribute Name}
  3. {$Parent:Attribute_Reference_0} (correct)
  4. {Parent_Name_0:Attribute_Name_0}
Reason

Your test-system expects the explicit {$Parent:Attribute_Reference_0} syntax to reference parent attributes on related products.

Q2. Which of the following is an example of correct syntax for a Dynamic Lookup?

  1. {PriceQuery:Monthly_Price__c}
  2. $Lookup(PriceQuery).Monthly_Price__c
  3. {Lookup(PriceQuery):Monthly_Price__c} (correct)
  4. {Lookup(PriceQuery).Monthly_Price__c}
Reason

Memorise the curly-brace + parentheses + colon form your testing platform uses.

Q3. $Linked is used for:

  1. provides a reference to Basket field
  2. used to get value of linked configuration
  3. provides a reference to the parent $Object of current basket (correct)
  4. None of the above
Reason

Test-system interpretation: $Linked references the parent object in the basket hierarchy.

Q4. Which attribute-based pricing mechanism should usually be used only for demos and prototyping?

  1. base price
  2. list price
  3. calculated price (correct)
  4. rule-based pricing
Reason

Calculated price is dynamic and often used for temporary/demo scenarios rather than stable production pricing.

Q5. Which of these Attribute Types cannot be overridden in a custom screen flow? (Choose two)

  1. lookup (correct)
  2. related product (correct)
  3. multiselect list
  4. checkbox
Reason

Lookup and related product attributes are system-driven references and not override-friendly in screen flows.

Q6. What are the business-specific objects of CloudSense?

  1. product definitions
  2. pricing models
  3. business processes
  4. orders
  5. all of the above (correct)
Reason

CloudSense models real business objects (products, pricing, processes, and order entities).

Q7. Which concept is useful for a family of products who share similar technology?

  1. product platform (correct)
  2. pricing platform
  3. process platform
  4. none of these
Reason

A product platform provides a common architecture and reusable components across a product family.

Q8. Which of the following is an example of correct syntax for a dynamic lookup? (alternate formatting in test)

  1. {PriceQuery:Monthly_Price__c}
  2. $Lookup(PriceQuery).Monthly_Price__c
  3. {Lookup(PriceQuery):Monthly_Price__c} (correct)
  4. {Lookup(PriceQuery).Monthly_Price__c}
Reason

Again, follow the test-system's curly braces + parentheses + colon pattern.

Q9. What action type would you use when you want to calculate a discounted amount?

  1. set discount price in combination with set line item price
  2. set discount type in combination with set discount value (correct)
  3. set value; set price actions
  4. set discount type in combination with set list price
Reason

Define the discount type (percent/fixed) and the value so the system computes the discount correctly.

MCQs — BSS / OSS / Telecom Enterprise (grouped)

Q1. Which system generates specific tasks for other systems that must be completed to activate service on the network?

  1. Customer Relationship Management (CRM)
  2. Order Handling
  3. Workflow Management (correct)
  4. Inventory
  5. Engineering
  6. Activation
Reason

Workflow Management orchestrates and creates tasks across systems to complete service activation.

Q2. Which of the following functions come under BSS and not OSS?

  1. order management
  2. inventory
  3. service assurance
  4. billing
  5. operations A and D (order management & billing) (correct)
  6. operations B and C
Reason

BSS covers customer-facing operations such as orders and billing; OSS covers inventory and service assurance.

Q3. Identify the Telecom Sales Console features (choose best option):

  1. Intuitive, configurable UI for product configuration
  2. Integrated with Salesforce
  3. Support for offers with multiple products
  4. Option A only
  5. Option A, B, and C (correct)
Reason

A modern telecom sales console is configurable, integrated with CRM, and supports multi-product offers/bundles.

Q4. Which of the following are Telecom line of business models NOT a business model?

  1. Retail
  2. Wholesale
  3. Enterprise
  4. Interconnect (correct)
  5. MVNO
  6. None of the above
Reason

Interconnect is a technical arrangement between networks, not a standalone business model.

Q5. Which of the following are advantages of a pricing model?

  1. does not require IT for modifying the price matrix
  2. business user/product manager can manage the changes to the matrix
  3. option a only
  4. option b only
  5. options a and b (correct)
Reason

Good pricing models empower business users to change pricing without IT involvement.

Q6. Which is the most important performance dimension for a product development project?

  1. time to target
  2. time to consumer
  3. time to seller
  4. time to market (correct)
Reason

Time to market measures how quickly a product becomes available to customers — critical for competitive advantage.

Q7. Which of the following is NOT supported in SS7?

  1. SMS
  2. Number portability
  3. Prepaid billing
  4. VOD (Video On Demand) (correct)
Reason

SS7 is a signaling protocol for telephony functions (SMS, roaming, billing) — VOD is a content delivery service outside SS7's scope.

Mnemonics & Quick Tips

  • Parent attribute: Think {$Parent:...} — parent precedes colon.
  • Lookup: Curly braces + parentheses + colon for test: {Lookup(Query):Attr}
  • $Linked: "Linked to parent object" — remember it as the basket's parent pointer.
  • Pricing: Demo = Calculated price; Production = Rule-based / List / Base + modifiers.
